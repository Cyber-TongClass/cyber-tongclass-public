# Updates 页面与受众标签设计

## 目标

新增 `/updates` 统一动态页面，在不新建独立内容库的前提下复用通班现有的 Convex `news` 与 `events` 数据。管理员可为新闻和活动维护受众范围与自定义分类标签，访客可在统一页面中筛选。

## 数据模型

新闻与活动记录新增两个可选字段：

- `audiences`：`["undergraduate", "graduate", "teacher"]` 的非空数组。一个内容可面向多个受众。
- `tags`：管理员维护的非空字符串数组，用作自由分类与前台筛选。

前台的“全部”只是筛选器的默认状态，不会作为数据库值保存。为兼容已存在的标签数据，迁移和编辑流程均保留 `tags`，不清空、不改名。

## 内容维护

新闻与活动的新增、编辑表单都提供相同的标签区域：一个多选复选框组用于本科生、研究生和老师；一个可添加、删除标签的文本输入用于自定义分类。提交时会去除空白、首尾空格与重复标签。

## 旧数据迁移

提供一个独立、由维护者手动触发且可重复运行的迁移脚本。该脚本仅为缺少 `audiences` 的既有新闻和活动写入三个受众值（本科生、研究生、老师），使其在“全部”及每种受众筛选下均可见。已存在 `audiences` 或 `tags` 的记录不被覆盖。迁移不会挂接到开发、构建或启动命令。

## `/updates` 页面

页面分别通过现有 React API hooks 读取已发布新闻和活动，再在浏览器中合并为时间线：新闻按 `publishedAt`，活动按开始日期排序。每张卡片标识内容类型并显示受众与自定义标签，点击仍通往原有新闻或活动详情。

筛选栏包括：

- 受众范围：全部、本科生、研究生、老师；可多选。选择多项时，保留命中任一所选受众的内容。
- 自定义分类：从当前加载内容的 `tags` 动态去重生成；可多选。选择多项时，保留命中任一分类的内容。
- 当受众和分类均有筛选时，内容必须同时满足两组条件。

原 `/news` 与 `/events` 页面保持既有功能和布局，不在本次改动范围内新增筛选。

## 错误处理与兼容性

新字段均为可选，以便旧记录在迁移前仍能正常读取。没有自定义标签的记录会保留在“全部”与受众筛选结果中，但不会匹配任意自定义分类。加载中、无结果和缺失字段场景均应有稳定的前台呈现。

## 验证

为筛选归一化与匹配逻辑先编写 Node 原生测试，依次验证受众、自由分类、两组组合条件、空标签及旧记录兼容性。再运行该测试与仓库 ESLint；如本地 Convex 环境可用，手动验证管理员标签编辑、迁移的幂等性和 `/updates` 筛选交互。
